O **TypeScript** é uma linguagem criada pela Microsoft que funciona como um **“superconjunto” do JavaScript**.

Isso significa que:

👉 Todo código JavaScript é válido em TypeScript  
👉 Mas o TypeScript adiciona recursos extras — principalmente **tipagem estática**

### Exemplo simples

JavaScript:
```javascript
function soma(a, b) {
  return a + b;
}
```

TypeScript:

```typescript
function soma(a: number, b: number): number {
  return a + b;
}
```

Aqui entra o grande diferencial:

- Em JS → você pode passar qualquer coisa
- Em TS → você define **o tipo dos dados** (number, string, boolean, etc.)

## Por que isso é importante?

Porque o TypeScript ajuda a evitar erros antes mesmo de rodar o código.

Exemplo de problema comum em JavaScript:
```javascript
soma(10, "20"); // resultado: "1020" 😬
```

No TypeScript:

```typescript
soma(10, "20"); // ❌ erro antes de executar
```

Ou seja:  
👉 Menos bugs  
👉 Código mais previsível  
👉 Melhor para projetos grandes

## Como o TypeScript surgiu?

O TypeScript foi criado por Anders Hejlsberg (sim, o mesmo criador do C#) dentro da Microsoft.
### Contexto da época (por volta de 2012):

- O JavaScript estava crescendo MUITO rápido
- Aplicações web estavam ficando grandes e complexas
- Faltavam ferramentas para organizar melhor o código

Problemas que existiam:

- Dificuldade de manutenção
- Muitos erros em tempo de execução
- Falta de estrutura para projetos grandes

A solução foi criar uma linguagem que:

- Mantivesse a essência do JavaScript
- Mas adicionasse **segurança e organização**

Assim nasceu o TypeScript.
## Como o TypeScript funciona na prática?

O TypeScript **não roda diretamente no navegador**.

Ele precisa passar por um processo chamado:

**Transpilação**

Ou seja:
```bash
TypeScript (.ts) → JavaScript (.js)
```

Quem faz isso é o compilador do TypeScript (`tsc`).
## Principais vantagens do TypeScript

✔ Tipagem estática  
✔ Melhor autocomplete nas IDEs  
✔ Código mais organizado  
✔ Facilita trabalho em equipe  
✔ Escala melhor em projetos grandes

## Mas nem tudo são flores…

Também tem alguns pontos de atenção:

- Curva de aprendizado inicial
- Precisa configurar ambiente (tsconfig, build, etc.)
- Mais “verboso” que JavaScript puro

## Resumindo de forma simples

Se fosse explicar em uma frase:

👉 **TypeScript é JavaScript com superpoderes para evitar erros e organizar melhor o código.**
